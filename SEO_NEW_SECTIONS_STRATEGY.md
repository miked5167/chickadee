## 🚀 Implementation Strategy

### Phase 1: Hockey Training Section (Weeks 1-4)

**Priority:** ⭐⭐⭐⭐⭐ HIGHEST

**Why Start Here:**
- Highest search volume (7,890/month = 54.8% of total)
- Strong local intent perfect for directory model
- ALL 8 cities show low competition
- Lowest implementation complexity (similar to advisors)

**Week 1-2: Database & Backend**
1. Create training_centers table in Supabase
2. Define data schema:
   - Name, location (address, city, state, coordinates)
   - Services (power skating, goalie training, skills development, etc.)
   - Programs (youth, junior, adult, goalie-specific)
   - Coaches/instructors
   - Pricing information
   - Schedule/availability
   - Photos, videos
   - Reviews and ratings
3. Create API endpoints for training centers
4. Implement search/filter functionality

**Week 3: Frontend Pages**
1. `/training` - Main listing page with search/filters
2. `/training/[slug]` - Individual training center detail pages
3. `/training/[city]` - City-specific landing pages (Toronto, Boston, etc.)
4. Implement map integration for location-based search

**Week 4: SEO Optimization**
1. Optimize meta titles/descriptions for training keywords
2. Add Schema.org LocalBusiness markup to all training centers
3. Create city-specific content for top 3 cities
4. Submit new pages to Google Search Console
5. Build internal links from homepage and footer

**Initial Content Strategy:**
- Import/add 20-30 training centers in top 3 cities (Toronto, Boston, Minneapolis)
- Focus on quality over quantity initially
- Encourage reviews from existing network

**Expected Results (Month 3):**
- 200-400 organic visits/month to training section
- Rankings for "[city] hockey training" keywords
- 5-10 qualified leads/month

## 🛠️ Technical Development Priorities

### Database Schema Additions

**training_centers table:**
```sql
CREATE TABLE training_centers (
  id UUID PRIMARY KEY,
  name TEXT NOT NULL,
  slug TEXT UNIQUE NOT NULL,
  description TEXT,
  address TEXT,
  city TEXT,
  state TEXT,
  country TEXT,
  postal_code TEXT,
  latitude DECIMAL,
  longitude DECIMAL,
  phone TEXT,
  email TEXT,
  website_url TEXT,
  logo_url TEXT,
  services TEXT[], -- ["power skating", "skills training", "goalie training"]
  age_groups TEXT[], -- ["youth", "junior", "adult"]
  price_range TEXT,
  hours JSONB,
  is_verified BOOLEAN DEFAULT false,
  is_claimed BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

**tournaments table:**
```sql
CREATE TABLE tournaments (
  id UUID PRIMARY KEY,
  name TEXT NOT NULL,
  slug TEXT UNIQUE NOT NULL,
  organizer TEXT,
  description TEXT,
  location_city TEXT,
  location_state TEXT,
  location_venue TEXT,
  start_date DATE,
  end_date DATE,
  registration_deadline DATE,
  age_divisions TEXT[], -- ["8U", "10U", "12U", "14U", "16U", "18U"]
  skill_levels TEXT[], -- ["A", "AA", "AAA", "Elite"]
  format TEXT, -- "round-robin", "playoff", "showcase"
  entry_fee TEXT,
  registration_url TEXT,
  website_url TEXT,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

**prep_schools table:**
```sql
CREATE TABLE prep_schools (
  id UUID PRIMARY KEY,
  name TEXT NOT NULL,
  slug TEXT UNIQUE NOT NULL,
  description TEXT,
  address TEXT,
  city TEXT,
  state TEXT,
  country TEXT,
  latitude DECIMAL,
  longitude DECIMAL,
  phone TEXT,
  email TEXT,
  website_url TEXT,
  logo_url TEXT,
  school_type TEXT, -- "boarding", "day", "both"
  program_levels TEXT[], -- ["varsity", "jv", "prep"]
  tuition_range TEXT,
  financial_aid_available BOOLEAN,
  enrollment_size INTEGER,
  student_teacher_ratio TEXT,
  college_placement_rate DECIMAL,
  hockey_facility_details TEXT,
  academic_info TEXT,
  is_verified BOOLEAN DEFAULT false,
  is_claimed BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

## 🚨 Risks & Mitigation Strategies

### Risk 1: Content Scaling Challenge

**Risk:** Manually adding hundreds of training centers, tournaments, and schools is time-consuming.

**Mitigation:**
1. Start with top markets only (80/20 rule)
2. Build tools for batch import from public sources
3. Encourage businesses to claim/submit their own listings
4. Partner with organizations (USA Hockey, tournament organizers, etc.)
5. Consider hiring VA for data entry at scale

### Risk 2: Keeping Tournament Data Current

**Risk:** Tournament dates change, some get cancelled - stale data hurts credibility.

**Mitigation:**
1. Build update workflow for tournament organizers
2. Email reminders to verify/update tournament info quarterly
3. Flag tournaments with past dates
4. Partner with major tournament companies for API access
5. Community reporting feature for outdated info

### Risk 3: Competition Emerges

**Risk:** Competitors see our success and launch similar directories.

**Mitigation:**
1. Move fast - first mover advantage is significant
2. Build moat with comprehensive data and reviews
3. Establish partnerships early (exclusive listings, featured placements)
4. Focus on user experience and site quality
5. Build brand recognition in hockey community

### Risk 4: Low Conversion Rates

**Risk:** Traffic comes but doesn't convert to leads/revenue.

**Mitigation:**
1. A/B test CTAs and form placements
2. Offer value-adds (comparison tools, guides, etc.)
3. Implement lead magnets (downloadable guides, checklists)
4. Retargeting campaigns for visitors
5. Email capture for nurture sequences

### Risk 5: Seasonality

**Risk:** Hockey has strong seasonal patterns - traffic may drop in summer.

**Mitigation:**
1. Tournaments and summer camps help fill summer gap
2. Prep school research happens year-round
3. Off-season training is actually busy period
4. Build email list during peak season
5. Content marketing for off-season planning
